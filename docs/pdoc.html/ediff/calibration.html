ediff.calibration

Module: ediff.calibration

The calibration of electron diffraction patterns:

  • The original diffraction pattern shows intensities as a function of distance-in-pixels.
  • The calibrated diffraction pattern shows intensities as a function of distance-in-q-vectors.
  • The term distance-in-q-vectors means that the distances = magnitudes of q-vector in [1/A].
  • This module gives the calibration constant, which converts distance-in-pixels to distance-in-q-vectors.
  • The final conversion is very simple: distance_in_q = distance_in_pixels * calibration_constant

Various ways how to determine the calibration constant:

  • The calibration constant can be calculated or determined by various functions in this module.
  • The calculations/determinations are not difficult, but we need at least one of the following:
    • ELD and XRD profiles => see ediff.calibration.Calculate.from_max_peaks function
    • some microscope parameters => see ediff.calibration.Calculate.from_microscope_parameters function
    • a pre-calibrated microscope => see the documentation at the begiining of ediff.calibration.Microscopes dataclass
  • Examples in this documentation:
    • Calculation of calibration constant for an arbitrary microscope => see ediff.calibration.Calculate class.
    • Determination of calibration constant for a known microscope => see ediff.calibration.Microscopes class.
    • Real life examples can be found in EDIFF website https://mirekslouf.github.io/ediff/docs

126    def from_max_peaks_in_range(
127            eld_profile, xrd_profile,
128            eld_range=None, xrd_range=None, messages=True):
129        '''
130        Calibration constant from the *selected* peaks on ED and PXRD profiles.
131        
132        * ELD/XRD profiles represent 1D electron/X-ray diffraction patterns.
133        * More info about ELD/XRD profiles in EDIFF
134          => see the docs of ediff.io.read_profile function.
135        * The peaks are selected using arguments *eld_range* and *xrd_range*.
136            - Both arguments are tuples of two floats = x-ranges.
137            - Only the maximal peaks in given ranges are considered.
138            - ED range is in [pixels] and PXRD range is given in [q-vectors].
139
140        Parameters
141        ----------
142        eld_profile : str or numpy.array
143            The *eld_profile* is
144            an electron diffraction profile in EDIFF format.
145            It can come as file (if *eld_profile* = str = filename)
146            or array (if *eld_profile* = numpy.array).
147        xrd_profile : str or numpy.array
148            The *xrd_profile* is
149            an X-rayd diffraction profile in EDIFF format.
150            It can come as file (if *xrd_profile* = str = filename)
151            or array (if *xrd_profile* = numpy.array).
152        eld_range : tuple of two floats, optional, default is None
153            The x-range in 1D ED profile,
154            in which we should search for the maximal peak.
155            The ED x-range is given in [pixels].
156        xrd_range : tuple of two floats, optional, default is None
157            The x-range in 1D XRD profile,
158            in which we should search for the maximal peak.
159            The XRD x-range is given in [q-vectors].
160        messages : bool, optional, default is True
161            If *messages* = True,
162            print some information
163            and the final calibration constant to stdout.
164            
165        Returns
166        -------
167        calibration constant : float
168            The multiplicative constant that converts
169            ED-profile X-coordinate-in-pixels
170            to X-coordinate-in-q-vectors [1/A].
171
172        Technical notes
173        ---------------
174        * ELD profile = 1D radially averaged
175          powder electron diffraction pattern
176            - in EDIFF, it is obtained from an experimental 2D difractogram
177        * XRD profile = 1D powder X-ray diffraction pattern
178            - in EDIFF, it is calculated from a standard CIF file
179              = Crystallographic Information File
180        * EDIFF format of ELD and XRD profiles employed in EDIFF package
181            - ELD and XRD profiles can come in the form of files or np.arrays
182            - Columns in files <=> rows in np.arrays (we use: *unpack=True*)
183            - ELD profile = 3 cols = pixels, intensity, bkgr-corrected-intsty
184            - XRD profile = 4 cols = 2theta[deg], S[1/A], q[1/A], norm-intsty
185        * EDIFF calculation of ELD and XRD profiles is best seen from examples:
186            - https://mirekslouf.github.io/ediff/docs -> worked example
187           '''
188        
189        # (1) Read ED and XRD diffraction profiles.
190        # * The profiles are supposed to be either filenames or numpy.arrays
191        # * In any case, the filenames or arrays should be in EDIFF format:
192        #   ED  = 3 columns: pixels, intensity, bkgr-corrected intensity
193        #   XRD = 4 columns: 2theta[deg], S[1/A], q[1/A], normalized-intensity
194        eld = ediff.io.Profile.read(eld_profile)
195        xrd = ediff.io.Profile.read(xrd_profile)
196            
197        # (2) Determine ranges, in which we search for peaks.
198        # (We search peaks either in the whole x-ranges
199        # (or in sub-ranges, if the args eld_range and xrd_range are given.
200        # ! X-ranges for ED and XRD are given in [pixel] and [q], respectively.
201        if eld_range is None:
202            e_range = eld 
203        else:
204            emin,emax = eld_range
205            e_range = eld[:,(emin<=eld[0])&(eld[0]<=emax)]
206        if xrd_range is None:
207            x_range = xrd
208        else:
209            xmin,xmax = xrd_range
210            x_range = xrd[:,(xmin<=xrd[2])&(xrd[2]<=xmax)]
211        
212        # (3) Get the peak/maximum value for ED and XRD.
213        # (The ranges where to search for peaks were determined in prev.step.
214        max_eld = float(e_range[0,(e_range[2]==np.max(e_range[2]))])
215        max_xrd = float(x_range[2,(x_range[3]==np.max(x_range[3]))])
216        print(f'Position of max.peak in ELD-profile: {max_eld:6.4f} in d[pix]')
217        print(f'Position of max.peak in XRD-profile: {max_xrd:6.4f} in q[1/A]')
218        
219        # (4) Calculate calibration constant
220        # (the constant converts d[pixels] to q[1/A]
221        calibration_constant = max_xrd/max_eld
222        print(f'Calibration constant: {calibration_constant:.6f}')
223        
224        # (5) Return the calculated calibration constant
225        return(calibration_constant)

228    def from_microscope_parameters(
229        voltage_kV, camera_length_mm, camera_pixel_size_um, binning,
230        messages=True):
231        '''
232        Calibration constant from microscope-specific parameters.
233        
234        * The calibration constant can be estimated from parameters,
235          which are typical of given microscope + camera system.
236        * The parameters we need to know are: (i) accelerating_voltage,
237          (ii) camera_length, (iii) camera_pixel_size, and (iv) binning.
238        * Warning: All three parameters may change
239          from experiment to experiment (although it is not typical);
240          detailed explanation is given below in Technical notes section.
241
242        Parameters
243        ----------
244        voltage_kV : int
245            Accelerating voltage in [kV].
246        camera_length_mm : float
247            Camera lenght in [mm].
248        camera_pixel_size_um : float
249            Camera pixel size in [um].
250        binning : int
251            Binning=1,2,4,... increases camera_pixel_size (1x,2x,4x,...).
252        messages : bool, optional, default is True
253            If {True}, the function prints outputs to stdnout.
254
255        Returns
256        -------
257        calibration constant : float
258            The multiplicative constant that converts
259            ED-profile X-coordinate-in-pixels
260            to X-coordinate-in-q-vectors [1/A].
261        
262        Technical notes
263        ---------------
264        * This calculation is based on four microscope parameters,
265          namely: (i) accelerating_voltage, (ii) camera_length_mm,
266          (iii) camera_pixel_size, and (iv) binning.
267        * Nevertheless, three of the parameters may change
268          from experiment to experiment:
269            - *accelerating_voltage_kV* for given TEM
270              can be changed by user (although it is not typical).
271            - *camera_length_mm* is routinely adjusted
272              to see the desired range of difractions;
273              the TEM software usually displays some information
274              about the nominal camera length.
275            - *binning* averages the neighboring pixels of the camera;
276              For example, the binning=1,2,4,...
277              increases the pixel size 1x,2x,4x,..
278        * IMPORTANT: The nominal camera length
279          shown by the microscope (*D*)
280          may not be the correct camera lenght (*camera_length_mm*)
281          needed for the calculation.
282            - Possible reasons: Exact amera position (bottom x upper camera),
283              projective lenses (which may change the magnification
284              of the diffraction pattern.
285            - Real-life solution: we need to know real *camera_length_mm*
286              for each theoretical/nominal/TEM-software-displayed
287              cammera length *D*!
288        '''
289        
290        # (1) Calculate relativistic wavelength of electrons
291        # (this is needed to convert CL to CC in the next step
292        Lambda = Utils.electron_wavelength(voltage_kV)
293        
294        # (2) Determine CL and CC
295        # * CL = CameraLenght
296        #   CL should be known from exp => argument camera_length_mm
297        # * CC = Camera Constant
298        #   CC = CL * Lamda => CC[mmA] = CL[mm]*Lamda[A]
299        #   The CC-CL => Camera equation: R*d = Lambda*CL = CC
300        CL = camera_length_mm
301        CC = camera_length_mm * Lambda
302        
303        # (3) Calculate final calibration constants.
304        # * The final calibration constants are:
305        #   R_calibration = pixel size in [mm]
306        #   S_calibration = k_calibration = pixel size in S [1/A]
307        #   q_calibration ............... = pixel size in q = 2*pi*S [1/A]
308        # * To calculate all three constants, we need just two parameters:
309        #   camera_pixel_size  = real size of the camera pixels
310        #   CC = camera_length = camera length, determined above
311        # * The calculation is easy
312        #   but it is performed in a separate function,
313        #   because it is employed also in calibration.Microscopes class.
314        # * The function contains
315        #   more details, including full justification of the calculations.
316        R_calibration, S_calibration, q_calibration = \
317            Utils.calc_final_calibration_constants(
318                camera_pixel_size_um = camera_pixel_size_um,
319                binning = binning,
320                camera_constant_mmA = CC)
321        
322        # (4) Print the calculated values if requested
323        if messages:
324            Utils.print_final_calibration_constants(
325                Lambda, CL, CC,
326                R_calibration, S_calibration, q_calibration)
327
328        # (5) Return the calculated calibration constant
329        # (We want the calibration constant in q [1/A]
330        return(q_calibration)

544    def electron_wavelength(U):
545        '''
546        Calculate relativistic wavelenght of accelerated electrons.
547    
548        Parameters
549        ----------
550        U : float
551            Accelerating voltage [kV].
552    
553        Returns
554        -------
555        Lambda : float
556            Relativistic wavelenght of electrons,
557            which were accelerated with the voltage *U*[kV]
558            in an electron microscope.
559        '''
560        # The formula below gives Lambda = wavelength in [A]
561        # (justification: textbooks of physics
562        # (my source: M:/MIREK.PRE/0_EM.PY/1_ELN-IN-EM/1eln_v-wl_v2.nb.html
563        
564        # Convert U from [kV] to [V]
565        U = U * 1000
566        
567        # Collect constants from scipy
568        h = scipy.constants.h
569        c = scipy.constants.c
570        e = scipy.constants.e 
571        m = scipy.constants.m_e
572        
573        # Calculate and return relativistic wavelenght of electrons
574        Lambda = h/np.sqrt(2*m*e*U) * (1/(np.sqrt(1+(e*U)/(2*m*c**2)))) * 1e10
575        return(Lambda)

579            camera_pixel_size_um, binning, camera_constant_mmA):
580        '''
581        Calculate final calibration constants

580            camera_pixel_size_um, binning, camera_constant_mmA):
581        '''
582        Calculate final calibration constants
583        from known CC and camera_pixel_size.
584
585        * Once we know *CC*, *camera_pixel_size*, and *binning*,
586          the calculation of the final calibration constants
587          is surprisingly easy.
588        * The full justification of the calculation
589          is given in the comments of the source code below.
590        
591        Parameters
592        ----------
593        camera_constant_mmA : float
594            The camera constant (from R*d = CL*Lambda = CC) in [mmA].
595        camera_pixel_size_um : float
596            The real dimension of one pixel of the camera (detector) in [um].
597        binning : int
598            Binning = 1,2,4,... increases pixel size (1x,2x,4x,...)
599
600        Returns
601        -------
602        R_calibration, S_calibration, q_calibration : three floats
603            * R_calibration = the camera pixel size in [mm].
604            * S_calibration = the camera pixel size in S-vector units [1/A].
605            * q_calibration = the camaera pixel size in q-vector units [1/A].
606        '''
607
608        # What do we want to calculate?
609        #  => three calibration constants = pix_size, S-units, q-units        
610        # Brief justification:
611        #  (a) Camera Equation: R*d = CL*Lambda = CC
612        #  (b) Bragg's Law: 2*d*sin(theta) = Lamda => S*d = 1 => S = 1/d
613        #  (c) Combine (a)+(b) for a known R[um] => here: R = size of 1 pixel
614        #      in general : R*d = CC => S = 1/d = R/CC
615        #      here/below : S_calibration[1/A] = R_calibration[mm] / CC [mm*A]
616        # -----
617        # Note: camera_pixel_size is influenced by binning
618        
619        # Calculate the calibration constants acc.to justification above
620        R_calibration = camera_pixel_size_um * binning / 1000
621        S_calibration = R_calibration / camera_constant_mmA
622        q_calibration = 2*np.pi * S_calibration
623
624        # Return the three calibration constants
625        return(R_calibration, S_calibration, q_calibration)
